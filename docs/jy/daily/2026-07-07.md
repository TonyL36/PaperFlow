## 验证结果

- 本地单元与模块验证：
  - `mvn -pl backend/services/api-gateway "-Dtest=EndpointAccessPolicyTest,JwtAuthGlobalFilterTest,RateLimitGlobalFilterTest" test` 通过
  - `mvn -pl backend/services/api-gateway test` 通过
- 启动验证：
  - `mvn -pl backend/services/api-gateway spring-boot:run` 可正常启动，`/actuator/health` 返回 `200 {"status":"UP"}`
  - `GET /api/v1/notifications` 在无 Bearer Token 时返回 `401 AUTH_MISSING_TOKEN`，说明受保护接口鉴权拦截正常
  - 在未启动 `user-service` 时，`GET /api/v1/public/users/u_demo` 与 `GET /api/v1/oauth/qq/callback` 都会因为下游 `localhost:8081` 连接拒绝返回 `500`，但限流头已经分别落在 `180` 与 `120`，说明分类先于转发已生效
  - 继续启动 `user-service`（本地 H2）后，`GET /api/v1/public/users/u_demo` 通过网关返回真实业务 `404 RES_NOT_FOUND`，同时保留 `X-RateLimit-Limit: 180`
  - 继续启动 `user-service` 后，`GET /api/v1/oauth/qq/callback` 通过网关返回真实业务 `400 Bad Request`，同时保留 `X-RateLimit-Limit: 120`
  - 这说明新访问策略在运行态已经完成了“正确分类 + 正确转发”，之前的 `500` 仅是下游未启动导致
