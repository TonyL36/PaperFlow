## 关键决策

- 认证链路继续以网关统一鉴权为核心，业务服务仅信任网关注入的用户上下文
- 用户能力按“自助能力（/users/me）”与“管理员能力（/admin/**）”分层，避免权限边界模糊
- 前端采用“统一请求封装 + 页面级错误态兜底”策略，减少重复异常处理逻辑
- 本地运行优先脚本化与参数化，默认提供安全值，按需覆盖端口与外部依赖

## 问题与修复

### 1) 本地接口地址被环境变量污染

- 问题：前端请求偶发指向错误 API 地址，表现为统一错误码但定位困难
- 修复：在本地启动脚本中显式注入 API 基地址，避免继承历史环境变量

### 2) 认证相关接口上下文不一致

- 问题：部分接口在放行策略与鉴权策略之间存在边界不清，导致登出等流程不稳定
- 修复：收敛放行范围，仅对必要登录链路放行，其余接口统一走鉴权与用户头注入

### 3) Windows 下构建产物占用导致重打包失败

- 问题：历史 Java 进程占用 jar，触发重命名失败
- 修复：补充脚本化清理能力与端口占用处理，降低重复手工排障成本
