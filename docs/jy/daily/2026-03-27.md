# 2026-03-27 开发记录（PaperFlow）

## 今日目标

- 修复阅读页 AI 回答异常与提示词回显
- 统一 AI 问答/翻译接口，拆分 Pathfinder 与 Chat 职责
- 补齐前端 Markdown 渲染一致性与文档缺口

## 今日完成

### 1) 问题定位

- 复盘前端 AI 调用点后确认：`PostDetailPage` 与 `PaperPdfReaderPage` 都在复用 `apiGeneratePathfinderPlan`
- 复现场景后确认：`/api/v1/pathfinder/sessions/plan` 在聊天场景会返回“4 阶段闯关路径”文案，导致提示词出现在对话区
- 明确结论：不是 Markdown 组件问题，而是“接口职责错配”

### 2) 后端接口改造

- 新增统一对话接口：`POST /api/v1/ai/chat`
- 新增控制器与 DTO：
  - `AiChatController`
  - `AiChatRequest`
  - `AiChatResponse`
- 新增服务 `AiChatService`：
  - 复用 `PathfinderAiProperties` 配置
  - 复用邮箱映射密钥策略
  - 统一返回 `assistantMessage`
- 网关新增路由 `content-ai`，开放 `/api/v1/ai/**`

### 3) 前端接入改造

- 新增前端 API 封装：`apiAiChat(...)`
- 页面切换：
  - `PostDetailPage` 问答/翻译改为调用 `apiAiChat`
  - `PaperPdfReaderPage` 问答/翻译改为调用 `apiAiChat`
- `PathfinderPage` 保持规划接口，不做语义混用

### 4) AI 消息渲染一致性

- 盘点 AI 渲染点后，确认三个页面统一 assistant 走 `AiMarkdown`
- 移除 PDF 页打字机中间态纯文本分支，避免“有时 Markdown 不解析”的观感问题

### 5) 启动与超时优化（同日）

- 双击启动脚本改为“先 quick，失败再 build”，缩短日常启动等待
- AI 请求层支持自定义超时，问答接口超时阈值上调到 90s，降低 `SYS_TIMEOUT`
