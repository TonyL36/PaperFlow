# 20 项目需求、任务与方法定义（基于项目申请书与任务书）

本文将《项目申请书》和《项目实训任务书》中的目标、范围、指标与实施路径，整理为一份面向研发落地的项目定义文档。目标是把“立项材料中的叙述”转化为“可实现、可分工、可验收”的工程说明，供后续产品、架构、开发、测试和答辩统一参考。

## 1. 项目定义

### 1.1 项目名称

- 文海问津 PaperFlow
- 定位：基于多 Agent 协作的智能科研阅读与知识管理系统

### 1.2 项目目标

- 构建面向 AI 研究者的智能科研中枢。
- 通过五 Agent 协作形成“智能发现 -> 审核入库 -> 摘要增强 -> PDF 阅读与问答 -> 学习路径规划”的闭环。
- 解决学术文献场景中的三类核心问题：
  - 信息过载：论文更新快，传统订阅和搜索方式效率低。
  - 知识碎片化：上传、搜索、阅读、笔记、规划分散在多个工具中。
  - 被动消费：现有工具偏检索或偏推荐，缺乏持续策展和个性化学习规划。

### 1.3 系统边界

- 本项目聚焦于“个人科研知识库”的发现、筛选、阅读、问答与规划。
- 本项目当前语料源以 arXiv、用户上传 PDF 和系统规划/对话发现结果为主。
- 本项目不以“全网开放搜索引擎”或“通用办公平台”为目标。
- 本项目对外表现为 Web 应用，核心由前端界面、后端服务、向量检索、五 Agent 编排组成。

## 2. 角色与责任定义

### 2.1 五 Agent 定义

#### Scout

- 角色：侦察员。
- 职责：根据日常漫游、规划任务或兴趣发现信号执行论文搜索与候选召回。
- 输入：
  - 用户兴趣画像
  - 路径规划目标
  - Sage 兴趣发现信号
  - 漫游策略参数
- 输出：
  - 候选论文列表
  - 搜索来源与理由
  - 补全元数据的二次搜索结果

#### Curator

- 角色：策展人，也是知识库唯一入口。
- 职责：审核所有入库请求，执行去重、命名规范化、价值筛选、多样性控制与来源标注。
- 输入：
  - Scout 候选论文
  - 用户上传 PDF
  - Pathfinder 规划结果
  - Sage 发现结果
- 输出：
  - 审核通过/拒绝结果
  - 标准化文件名
  - 入库元数据
  - 来源标记：`roaming / uploaded / planned / discovered`

#### Editor

- 角色：编辑。
- 职责：对通过审核的论文生成结构化摘要、标签、卡片文案和展示素材。
- 输入：
  - 已审核通过的论文元数据与正文
- 输出：
  - 三段式结构化摘要
  - 关键词标签
  - 卡片 Teaser 文案
  - 可视化展示所需的内容摘要

#### Sage

- 角色：智者。
- 职责：面向 PDF 阅读器和知识星云提供问答服务，并根据用户提问模式识别潜在兴趣。
- 输入：
  - 当前 PDF 页上下文
  - 用户划词内容
  - 向量检索结果
  - 全局知识库上下文
- 输出：
  - 局部问答结果
  - 全局 RAG 问答结果
  - 引用来源
  - 兴趣发现信号

#### Pathfinder

- 角色：规划师。
- 职责：把用户目标转换为阶段式学习路径，并将路径目标反馈给 Scout 执行深度搜索。
- 输入：
  - 用户目标
  - 已有知识库
  - Sage 提供的能力缺口信息
- 输出：
  - 学习阶段定义
  - 每阶段论文清单
  - 阶段里程碑
  - 下一轮检索议程

### 2.2 用户侧角色

- 普通用户：
  - 上传论文
  - 阅读 PDF
  - 向 Sage 提问
  - 接收漫游推荐
  - 查看和执行学习路径
- 系统管理员或演示操作者：
  - 配置模型、索引和调度参数
  - 检查 Agent 流程
  - 验证接口与运行状态

## 3. 业务流程定义

### 3.1 四路入库流程

#### 漫游入库

- 触发：系统定时漫游。
- 流程：Scout 搜索 -> Curator 审核 -> Editor 增强 -> 入库 -> 出现在推荐流。
- 目标：保证每日有 3-5 篇合格内容进入用户视野。

#### 自主上传

- 触发：用户上传 PDF。
- 流程：上传 -> 解析 -> Curator 规范化 -> 向量化 -> 入库 -> 可在阅读器和知识库管理页查看。
- 目标：将“用户手头已有论文”快速并规范地纳入个人知识库。

#### 规划入库

- 触发：用户提出学习目标。
- 流程：Pathfinder 规划 -> Scout 深度搜索 -> Curator 审核 -> 标记为计划关卡 -> 入库。
- 目标：把“目标导向的系统学习”变成结构化任务。

#### 对话发现入库

- 触发：Sage 在对话中识别兴趣主题。
- 流程：Sage 发出发现信号 -> Scout 搜索 -> Curator 审核 -> 用户确认 -> 入库。
- 目标：把“自然对话中暴露的兴趣”转化为零侵扰知识采集。

### 3.2 阅读与问答流程

- PDF 上传后由解析服务提取文本块和公式块。
- 前端阅读器通过坐标映射获取用户划选区域。
- Sage 根据当前页文本块、公式块和检索结果生成回答。
- 若用户提问超出当前论文，Sage 走全局知识库 RAG。

### 3.3 学习路径流程

- 用户输入目标，例如“14 天学习扩散模型”。
- Pathfinder 输出阶段式学习计划。
- Scout 根据阶段目标搜索候选论文。
- Curator 审核并入库后，计划界面展示闯关进度。

## 4. 需求定义

### 4.1 功能需求

#### FR-1 多 Agent 编排

- 系统必须实现五 Agent 协作工作流。
- 系统必须支持节点化状态流转、条件路由和可追踪的执行历史。
- 系统必须支持从不同触发源进入统一工作流。

#### FR-2 统一策展与入库

- 所有论文入库必须经过 Curator 审核。
- 系统必须支持去重检测、文件名规范化和来源标记。
- 系统必须支持漫游、上传、规划、对话发现四种入库路径。

#### FR-3 文献搜索与候选召回

- Scout 必须支持至少一种真实搜索源。
- Scout 必须支持基于兴趣、目标和问题构建查询。
- Scout 必须支持缺失元数据时的补全机制。

#### FR-4 PDF 阅读与局部问答

- 系统必须支持 PDF 上传、异步解析与浏览器内阅读。
- 系统必须支持划词问答和全文问答。
- 系统必须支持公式与自然语言混合内容的处理。

#### FR-5 全局知识库与 RAG

- 系统必须对入库论文进行分块和向量化。
- 系统必须支持基于相似度检索的跨论文问答。
- 系统必须支持回答引用来源显示。

#### FR-6 知识星云可视化

- 系统必须支持论文向量降维后的二维可视化展示。
- 系统必须支持聚类着色、缩放、拖拽、悬停等交互。
- 系统应支持阅读轨迹叠加。

#### FR-7 学习路径与闯关

- 系统必须支持阶段式学习计划生成。
- 系统必须支持阶段目标、候选论文和里程碑定义。
- 系统应支持计划与漫游共享同一论文库。

### 4.2 非功能需求

#### NFR-1 可追踪性

- 每次 Agent 运行都应有唯一 `run_id`。
- 系统应保存状态快照、路由历史和关键消息。

#### NFR-2 一致性

- 入库接口应支持幂等。
- 同一论文重复接入不应产生重复记录。

#### NFR-3 性能

- PDF 阅读器在 100 页以内文档上应保持可接受的首屏加载时间。
- 向量检索应在交互可接受范围内返回结果。

#### NFR-4 可扩展性

- 模型供应商应可替换。
- 搜索源、Embedding 模型和 Agent 节点实现应可替换。

#### NFR-5 成本可控

- 高频任务应优先使用中等成本模型。
- 大批量向量化和摘要生成必须支持异步化与限流。

## 5. 任务拆解

### 5.1 Agent 编排任务

- 定义共享状态模型：
  - `messages`
  - `user_profile`
  - `goals`
  - `search_queries`
  - `candidate_pool`
  - `approved`
  - `rejected`
  - `learning_plan`
  - `answer`
- 定义节点：
  - `Scout`
  - `Curator`
  - `Editor`
  - `Sage`
  - `Pathfinder`
- 定义条件边：
  - 起始节点路由
  - Curator 后是否继续搜索
  - Editor 后是否进入 Sage 或 Pathfinder
  - Sage 后是否继续规划

### 5.2 检索与入库任务

- 对接 arXiv 或其他论文搜索源。
- 设计候选论文统一数据结构。
- 实现重复检测、来源标记和标准化文件名。
- 设计上传、规划、对话发现与漫游四条入库链路。

### 5.3 内容增强任务

- 生成结构化摘要。
- 提取标签和卡片文案。
- 规划展示所需的摘要字段和排序逻辑。

### 5.4 RAG 任务

- PDF 文本与公式提取。
- 语义分块。
- Embedding 生成。
- 向量库存储与相似度检索。
- 局部 RAG 和全局 RAG 组装。

### 5.5 前端与可视化任务

- PDF.js 阅读器。
- 划词定位和公式渲染。
- 知识星云散点图。
- 学习路径闯关界面。
- 漫游推荐卡片与详情页。

### 5.6 工程化任务

- API 定义与统一 Envelope。
- 异步任务与重试机制。
- 模型调用抽象层。
- 限流、超时和错误恢复。
- 文档与演示数据准备。

## 6. 方法定义

### 6.1 多 Agent 方法

- 方法目标：将复杂科研任务拆成职责清晰的专业节点。
- 方法核心：
  - 用共享状态对象承接上下文。
  - 用图编排定义节点和边。
  - 用条件路由表达不同业务触发场景。
- 当前推荐实现：
  - 使用 LangGraph 作为图编排层。
  - 为每个 Agent 提供清晰输入、输出和状态更新策略。

### 6.2 策展与多样性控制方法

- 去重方法：
  - 标题相似度匹配
  - 作者和年份联合判断
  - 预印本与正式版的同源合并
- 价值审核方法：
  - 过滤与用户目标低相关候选
  - 降低与既有知识库过度相似论文的优先级
- 甜蜜区策略：
  - 将“相关但不重复”的候选作为优先入库对象
  - 防止知识库过度同质化

### 6.3 问答与兴趣发现方法

- 局部问答方法：
  - 读取当前页文本块和公式块
  - 拼接划词上下文
  - 交给问答模型生成回答
- 全局 RAG 方法：
  - 将问题向量化
  - 检索 Top-K 相关 chunk
  - 结合引用元数据生成回答
- 兴趣发现方法：
  - 基于提问频次、语义聚类和显式表达识别主题偏好
  - 将兴趣信号反馈给 Scout 和 Curator

### 6.4 学习路径规划方法

- 输入：目标、兴趣、现有知识库。
- 输出：阶段、论文、里程碑、下一轮检索议程。
- 核心思想：
  - 先基础，再核心方法，再前沿问题
  - 用阶段化拆解替代一次性推荐

### 6.5 向量化与知识星云方法

- 文本提取：保留段落边界和公式信息。
- 分块策略：以段落为主，避免在句中切断。
- 向量化：使用文本 Embedding 模型生成语义向量。
- 检索：向量相似度召回，可叠加关键词重排。
- 可视化：使用 t-SNE 或 UMAP 降维到二维坐标，再由 D3.js 展示。

## 8. 验收定义

### 8.1 最小可用验收

- 五 Agent 工作流可运行。
- 至少一条真实搜索链路可用。
- 上传 PDF 后可解析并在前端阅读。
- Sage 可基于局部或全局上下文回答问题。
- Pathfinder 可生成阶段式学习计划。

### 8.2 进阶验收

- 四路入库全可运行。
- 向量知识库与知识星云可用。
- Curator 能稳定完成去重和命名规范化。
- Editor 生成可展示的论文卡片。

## 9. 当前实现与目标实现的差距

### 9.1 当前已具备

- LangGraph 五 Agent 工作流骨架。
- arXiv 搜索与本地语料兜底。
- Curator/Editor/Sage/Pathfinder 的规则版实现。
- PDF 上传、解析、阅读与划词翻译。
- FastAPI 接口与运行状态查询。

### 9.2 仍需补齐

- Curator 的模型化语义审核。
- Editor 的模型化摘要和标签生成。
- Sage 的真正向量 RAG。
- Pathfinder 的模型化路径规划。
- 向量库存储、Embedding 管线与知识星云可视化。

## 10. 推荐落地顺序

1. 固化数据模型和数据库 Schema。
2. 打通上传、解析、入库、向量化基础链路。
3. 用中等成本模型完善 Editor、Curator、Sage、Pathfinder。
4. 接入 Embedding 与向量检索。
5. 实现知识星云和学习路径 UI。
6. 最后补充兴趣发现和高级策展策略。
